1. Introduction

KodeView ("the App," "we," "our," "us") is an offline code viewer for public GitHub repositories, available on Android and iOS. This Privacy Policy describes how the App handles information when you install and use it.

KodeView is designed with a minimal data footprint. We do not operate our own servers, we do not create user accounts, and we do not track you. The only external service the App communicates with is GitHub's public API, and only on your explicit instruction (e.g. when you search for or download a repository).

This Privacy Policy explains how KodeView handles information when you use the app. If you do not agree, please uninstall the App. This policy should be read alongside GitHub's Privacy Statement , which governs any traffic the App sends to GitHub's infrastructure.

2. Information We Do Not Collect

KodeView does not collect, record, store, transmit, or share any of the following:

  • Name, email address, username, or any other personal identifier
  • Phone number, postal address, or IP address
  • Precise or approximate location data
  • Device advertising ID, IDFA, or hardware fingerprint
  • Contacts, photos, camera, or microphone data
  • Payment or financial information
  • Browsing history or in-app navigation events
  • Crash reports or diagnostic telemetry
  • Usage analytics (session length, feature taps, screen views)

We do not embed any analytics SDKs (e.g. Firebase Analytics, Mixpanel, Amplitude), advertising SDKs, or crash reporting SDKs. No data is sold, licensed, or shared with data brokers or advertisers.

3. Information Handled on Your Device

The following data is generated and stored locally on your device. It never leaves your device except as described in Section 4.

3.1 Downloaded Repository Files

When you browse or download a public GitHub repository, the App saves the repository's files in the App's private local storage (sandboxed by the operating system). This data is accessible only to KodeView and is not accessible to other apps. It is deleted when you remove the repository from the App or uninstall the App.

3.2 App Settings and Preferences

Your in-app preferences — such as colour theme, font size, reader layout, pinned repositories, and sort order — are saved in local app storage on your device. This data is never transmitted anywhere.

3.3 GitHub Personal Access Token (Optional)

You may optionally enter a GitHub Personal Access Token (PAT) in Settings to increase GitHub API rate limits or to access repositories you own. If you do:

  • The token is held in memory only for the duration of the current session.
  • It is never written to persistent storage (disk, keychain, or shared preferences).
  • It is transmitted only to GitHub's API (via HTTPS) and to no other destination, only when you initiate a request that requires authentication.
  • It is discarded immediately when you remove it in Settings or close the App.

We strongly recommend using a fine-grained token scoped to read-only access on public repositories. Never paste a token with write or admin permissions into any third-party app.

4. Information Sent to GitHub

KodeView communicates exclusively with GitHub's public infrastructure (api.github.com and associated download URLs such as codeload.github.com and raw.githubusercontent.com). The App sends requests on your behalf when you:

  • Search for a repository by name or URL
  • Browse a repository's file tree or view a file
  • Download or clone a repository for offline use
  • Refresh or re-sync a saved repository

These requests include:

  • The repository URL or search query you entered
  • Standard HTTP metadata (your device's IP address, User-Agent string, request timestamp)
  • Your optional PAT in the Authorization: Bearer header, only when you have provided one

GitHub collects and processes this traffic under their own Privacy Statement. KodeView has no control over GitHub's data practices. You can review GitHub's policy at:
https://docs.github.com/en/site-policy/privacy-policies/github-privacy-statement

KodeView is not affiliated with, endorsed by, or in any way officially connected to GitHub, Inc. or its parent company Microsoft Corporation.

5. App Permissions

KodeView requests only the permissions necessary for its core functionality:

Permission Why it is needed
Internet / Network access To send API requests to GitHub and download repository files.
Local storage (read/write) To save repository files and settings on your device for offline use.

No other device permissions are requested or used. The App does not access your camera, microphone, contacts, calendar, location, Bluetooth, NFC, or biometric sensors.

6. Data Retention and Deletion

Because we operate no servers, all data retention and deletion is in your hands:

  • Remove a repository: Tap the repository menu on the home screen and select Remove. All locally stored files for that repository are deleted immediately.
  • Remove your GitHub token: Go to Settings → GitHub Token and tap Remove Token. The token is cleared from memory immediately.
  • Reset all settings: Go to Settings → Advanced → Reset App Data to erase all preferences and pinned repositories.
  • Uninstall the App: Uninstalling KodeView permanently deletes all locally stored repositories, settings, and any in-memory data. There is no remote account or cloud backup to delete separately.

7. Security

All network communication between the App and GitHub occurs over HTTPS (TLS 1.2+). Certificate pinning is applied to GitHub's API endpoints to defend against man-in-the-middle attacks.

Your optional PAT is held in application memory only and is never written to disk, shared preferences, the device keychain, or any log file. Local repository files are stored in the App's private sandbox, which is protected by the operating system's app isolation model (Android: internal storage; iOS: App Sandbox).

No method of electronic storage or transmission is 100% secure. While we have designed the App to minimise the data it handles in the first place, we cannot guarantee absolute security against all threats. We encourage you to keep your device's operating system and the App updated to receive the latest security patches.

8. Third-Party Services

KodeView's only external dependency is GitHub's public API. The App does not integrate any of the following:

  • Analytics services (Firebase, Amplitude, Mixpanel, etc.)
  • Advertising networks or SDKs
  • Crash reporting services (Sentry, Crashlytics, etc.)
  • Cloud storage or sync services
  • Social login or OAuth providers (other than GitHub's API for rate-limit purposes)
  • Payment processors
  • AI or machine learning cloud APIs

If this changes in a future version, this policy will be updated before the new integration ships, and users will be notified as described in Section 11.

9. Children's Privacy

KodeView is not directed at children. Specifically:

  • The App is not designed for, marketed to, or intended to be used by children under 13 years of age (or the applicable minimum age in your jurisdiction, e.g. 16 in the EU under GDPR).
  • We do not knowingly collect personal information from children. Because we collect no personal information from anyone, we do not collect any from children either.
  • If you are a parent or guardian and believe your child has used the App in a way that may have resulted in data being transmitted to GitHub (e.g. through a search query), please contact us at the address in Section 12 and review GitHub's Privacy Statement for how that data may be handled by them.

10. Your Rights

Because KodeView does not collect, store, or process personal data on our own servers, most data subject rights (access, rectification, erasure, portability) are exercised directly on your device — you have full local control as described in Section 6.

For any data processed by GitHub on our behalf (e.g. API request logs tied to your IP address), please contact GitHub directly via their privacy portal: https://support.github.com/contact/privacy

If you are located in the European Economic Area (EEA), United Kingdom, or another jurisdiction with applicable privacy laws, you may have additional rights including the right to lodge a complaint with your local data protection authority. We are happy to assist where we can — please reach out via Section 12.
